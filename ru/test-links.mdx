---
title: 'Совместное использование тестов'
description: 'Как делиться тестами и получать результаты'
---

Вероятно, вы создаете тесты для того, чтобы делиться ими со своими студентами, коллегами или друзьями. Платформа Testsplatform предоставляет удобный способ поделиться тестами — ссылки. Ссылки позволяют вам создать тест и поделиться им с различными настройками.

Примеры сценариев использования:
1. Ссылка на тест с включенным режимом обучения. В этом случае пользователи будут видеть правильные ответы и объяснения сразу после ответа на вопрос. Это поможет им изучить новый материал.
2. Ссылка на тест с ограничением по времени и без ответов и объяснений поможет вам оценить знания ваших студентов.

Важно: Вы можете создать ссылку только после публикации теста.

Существует 2 основных сценария, которые мы можем рассмотреть при совместном использовании тестов:
1. Вы хотите поделиться тестом с кем угодно в интернете и вас не интересуют их результаты (вас может интересовать только статистика).
2. Вы хотите поделиться тестом с кем-то и видеть результаты этого человека.

### Сценарий 1
Рассмотрим первый сценарий. Для его реализации вам нужно создать ссылку с установленным флажком «Анонимно». Вы можете настроить остальные параметры по своему усмотрению, мы рассмотрим их позже. После создания ссылки вы можете поделиться ею в социальных сетях, через мессенджеры и т. д. Пользователи, у которых есть ссылка, смогут пройти тест и увидеть свои результаты (если вы разрешили это в настройках).

### Сценарий 2
Теперь рассмотрим второй сценарий. Для его реализации вам нужно создать ссылку и не выбирать флажок «Анонимно». В этом случае пользователи, прошедшие тест, не смогут увидеть свои результаты, но вы их увидите. Вы можете создать множество индивидуальных ссылок (даже с однократным использованием) или создать одну ссылку для группы людей (например, для класса).

### Настройки ссылки
При создании ссылки вы можете задать некоторые параметры. Рассмотрим их:

#### URL
Вы можете задать читаемый URL для вашего теста. Полная ссылка состоит из трех частей:
 - домен testsplatform.com
 - URL профиля (или домен организации, если тест принадлежит организации)
 - URL теста

**Будьте осторожны при изменении частей вашего URL, включая URL профиля в настройках учетной записи. После изменения URL тест будет недоступен по предыдущему URL.**

Также вы можете проигнорировать это поле и сгенерировать случайный короткий URL. В большинстве случаев это более удобно.

#### Количество использований
Если вы хотите ограничить количество использований ссылки, вы можете установить ограничение на использование. После достижения лимита ссылка будет отключена, и никто не сможет пройти тест через эту ссылку.

#### Порог прохождения
Вы можете установить проходной порог для теста в процентах. Если результат пользователя превышает порог, тест считается пройденным. В противном случае он считается не пройденным.

#### Ограничение по времени
Вы можете установить ограничение по времени для прохождения теста. Значение указывается в минутах. Значение 0 (по умолчанию) означает, что ограничения по времени нет.
Если есть ограничение по времени, пользователь увидит таймер во время теста. Когда таймер дойдет до 0, пользователь будет перенаправлен на страницу завершения теста
и не сможет больше отвечать на вопросы или обновлять свои оценки. Таймер продолжает работать, даже если пользователь обновляет страницу или закрывает и открывает ее снова.
Связанная настройка: «Приостанавливаемый», см. ниже.

#### Количество вопросов в тесте при прохождении
Чтобы избежать одинаковых вопросов в каждом тесте, вы можете установить меньшее количество для параметра «Количество вопросов в тесте», чем общее количество вопросов.
Таким образом, каждый раз, когда пользователь проходит тест, он будет получать случайный набор вопросов. Например, если ваш тест содержит 100 вопросов, и вы установили значение 25,
каждый пользователь увидит 25 случайных вопросов из 100.

#### Случайный порядок вопросов
Если включено, порядок вопросов будет разным каждый раз, когда тест будет проходиться. Это может помочь предотвратить списывание.

#### Случайный порядок вариантов ответов
Аналогично случайному порядку вопросов, эта настройка перемешивает варианты ответов внутри каждого вопроса каждый раз, когда тест проходится.
Будьте осторожны с этой настройкой, если ваши варианты помечены (A, B и т. д.), так как объяснения не будут обновляться в соответствии с перемешиванием.

#### Можно приостановить
Когда эта настройка активна, пользователь может приостановить тест, и таймер остановится. Пользователь может покинуть страницу и вернуться через долгое время, после чего он сможет
продолжить тест и продолжить отвечать на вопросы.

#### Режим обучения
Эта настройка позволяет немедленно показывать ответ на вопрос в тесте. Например, вы хотите использовать тест в учебных целях.
Вы создаете вопросы, добавляете объяснения, и затем, когда пользователь отвечает на вопрос и нажимает «Далее», он сразу видит, правильный ли его ответ, какой ответ правильный и почему (с объяснением).

#### Видимость ответов и результатов (доступно только для неанонимных тестов)
Вы можете контролировать, что пользователь видит после прохождения теста. Ниже представлена таблица, показывающая, что увидит пользователь в зависимости от значения настройки:

|                   |  Все   | Все, кроме объяснений | Статистика и правильность ответов | Только статистика | Ничего |
|        ---        |  ---   |           ---         |              ---                  |     ---           |   ---  |
| Статистика        |   +    |            +          |               +                   |      +            |    -   |
| Правильность      |   +    |            +          |               +                   |      -            |    -   |
| Правильные ответы |   +    |            +          |               -                   |      -            |    -   |
| Объяснения        |   +    |            -          |               -                   |      -            |    -   |


Если вы больше не хотите, чтобы ваш тест был доступен, вы можете архивировать ссылки. Без активных ссылок тест не может быть пройден. Однако пользователи, которые уже прошли тест, все еще смогут видеть его в своих оценках и просматривать вопросы, ответы и объяснения.
